# SkillTrack API

A RESTful API for managing an online learning platform, built with **Spring Boot 4** following a **Hexagonal Architecture** (Ports & Adapters) organized as **Vertical Slices**. The project demonstrates complex JPA/Hibernate entity mappings, clean architecture patterns, domain events, caching, and modern testing strategies.

## Architecture

The project follows **Hexagonal Architecture** (Ports & Adapters) organized into **Vertical Slices**. Each slice owns its full stack from domain to infrastructure. Cross-slice dependencies are only allowed at the outbound port (domain interface) level.

```
src/main/java/com/nsalazar/skill_track/
├── shared/
│   ├── exception/       # GlobalExceptionHandler, ResourceNotFoundException, BusinessValidationException
│   ├── persistence/     # Auditable base class (@CreatedDate / @LastModifiedDate)
│   ├── event/           # Domain events + async listener
│   └── scheduled/       # Scheduled tasks
├── student/
│   ├── domain/          # Student record + StudentRepositoryPort (outbound)
│   ├── application/     # Use case interfaces (inbound) + service implementations
│   └── infrastructure/
│       ├── in/web/      # StudentController, DTOs, MapStruct web mapper
│       └── out/persistence/ # StudentJpaEntity, JPA repository, adapter, MapStruct persistence mapper
├── profile/             # Same structure — 1:1 relationship with Student
├── instructor/          # Same structure
├── course/              # Same structure — N:1 relationship with Instructor
├── enrollment/          # Same structure — N:M bridge between Student and Course
└── review/              # Same structure — one review per student per course
```

**Dependency rule:**
```
Controller → UseCase (port/in) → Service → RepositoryPort (port/out) ← PersistenceAdapter → JpaRepository
```

A service in slice A may import `port/out` interfaces from slice B, but never application services or infrastructure classes from other slices.

## Entity Relationships

```
Student ──────────────── Profile
  │  (1:1, student owns)
  │
  │ (N:M via Enrollment)
  │
Student ──── Enrollment ──── Course ──── Instructor
  │                              (N:1)       (1:N)
  │
  └──── Review ────────────── Course
         (one per student/course pair)
```

| Relationship | Entities | JPA Mapping |
|---|---|---|
| One-to-One | Student → Profile | `@OneToOne @JoinColumn(name="student_id")` on `ProfileJpaEntity` |
| Many-to-One | Course → Instructor | `@ManyToOne @JoinColumn(name="instructor_id")` on `CourseJpaEntity` |
| One-to-Many | Instructor → Courses | `@OneToMany(mappedBy="instructor", cascade=ALL)` on `InstructorJpaEntity` |
| N:M (explicit) | Student + Course via Enrollment | `EnrollmentJpaEntity` with two `@ManyToOne` + unique constraint `(student_id, course_id)` |
| N:M (explicit) | Student + Course via Review | `ReviewJpaEntity` with two `@ManyToOne` + unique constraint `(student_id, course_id)` |

All primary keys are **UUID**, generated by the database via `@GeneratedValue(strategy = GenerationType.UUID)`.

All entities extend `Auditable`, which automatically fills `createdAt` / `updatedAt` via JPA Auditing.

## Configuration

The application reads database credentials from environment variables with fallback defaults:

| Variable | Default | Description |
|---|---|---|
| `DB_USERNAME` | `root` | MySQL username |
| `DB_PASSWORD` | `root` | MySQL password |

The database `skill_track_db` is created automatically on first startup (`createDatabaseIfNotExist=true`).

To override, set the environment variables before running:
```bash
export DB_USERNAME=myuser
export DB_PASSWORD=mypassword
```

## Running the Application

All commands are run from the `skill-track/` directory:

```bash
# Build (skipping tests)
./mvnw clean package -DskipTests

# Run the application
./mvnw spring-boot:run

# Run all unit tests (no Docker required)
./mvnw test -Dtest="*ServiceTest,*ControllerTest"

# Run all tests including Testcontainers integration tests (requires Docker)
./mvnw test
```

Once running:

| URL | Description |
|---|---|
| `http://localhost:8080` | API base |

| `http://localhost:8080/api-docs` | OpenAPI JSON |
| `http://localhost:8080/actuator/health` | Health check |
| `http://localhost:8080/actuator/metrics` | Metrics |

## API Endpoints

### Students — `/api/v1/students`

| Method | Path | Description | Status |
|---|---|---|---|
| `GET` | `/api/v1/students` | List all students (paginated) | 200 |
| `POST` | `/api/v1/students` | Create a student | 201 |
| `GET` | `/api/v1/students/{id}` | Get student by ID | 200 / 404 |
| `PATCH` | `/api/v1/students/{id}` | Partially update a student | 200 / 404 / 409 |
| `DELETE` | `/api/v1/students/{id}` | Delete a student | 204 / 404 |

### Profiles — `/api/v1/students/{studentId}/profile`

| Method | Path | Description | Status |
|---|---|---|---|
| `GET` | `/api/v1/students/{studentId}/profile` | Get profile for a student | 200 / 404 |
| `POST` | `/api/v1/students/{studentId}/profile` | Create profile for a student | 201 / 409 |
| `PATCH` | `/api/v1/students/{studentId}/profile` | Partially update a profile | 200 / 404 / 409 |
| `DELETE` | `/api/v1/students/{studentId}/profile` | Delete a profile | 204 / 404 |

### Instructors — `/api/v1/instructors`

| Method | Path | Description | Status |
|---|---|---|---|
| `GET` | `/api/v1/instructors` | List all instructors (paginated) | 200 |
| `POST` | `/api/v1/instructors` | Create an instructor | 201 |
| `GET` | `/api/v1/instructors/{id}` | Get instructor by ID | 200 / 404 |
| `PATCH` | `/api/v1/instructors/{id}` | Partially update an instructor | 200 / 404 / 409 |

### Courses

| Method | Path | Description | Status |
|---|---|---|---|
| `GET` | `/api/v1/courses` | List all courses (paginated) | 200 |
| `POST` | `/api/v1/instructors/{instructorId}/courses` | Create a course under an instructor | 201 / 404 |
| `GET` | `/api/v1/courses/{id}` | Get course by ID | 200 / 404 |
| `PATCH` | `/api/v1/courses/{id}` | Partially update a course | 200 / 404 / 409 |
| `DELETE` | `/api/v1/courses/{id}` | Delete a course | 204 / 404 |

### Enrollments — `/api/v1/students/{studentId}/enrollments`

| Method | Path | Description | Status |
|---|---|---|---|
| `GET` | `/api/v1/students/{studentId}/enrollments` | List all enrollments for a student | 200 / 404 |
| `POST` | `/api/v1/students/{studentId}/enrollments/{courseId}` | Enroll a student in a course | 201 / 404 / 409 |
| `PATCH` | `/api/v1/students/{studentId}/enrollments/{courseId}/progress` | Update progress (0–100%) | 200 / 404 / 409 |
| `DELETE` | `/api/v1/students/{studentId}/enrollments/{courseId}` | Unenroll a student from a course | 204 / 404 |

### Reviews

| Method | Path | Description | Status |
|---|---|---|---|

## Request / Response Examples

### Create Student
```http
POST /api/v1/students
Content-Type: application/json

{
  "firstName": "John",
  "lastName": "Doe",
  "email": "john.doe@example.com"
}
```

### Create Profile
```http
POST /api/v1/students/{studentId}/profile
Content-Type: application/json

{
  "bio": "Full-stack developer with 5 years of experience",
  "linkedInUrl": "https://linkedin.com/in/johndoe",
  "phoneNumber": "+1234567890",
  "githubUrl": "https://github.com/johndoe",
  "portfolioUrl": "https://johndoe.dev",
  "avatarUrl": "https://avatars.example.com/johndoe.png"
}
```

### Create Instructor
```http
POST /api/v1/instructors
Content-Type: application/json

{
  "firstName": "Jane",
  "lastName": "Smith",
  "email": "jane.smith@example.com",
  "bio": "Java architect and Spring Boot expert"
}
```

### Create Course
```http
POST /api/v1/instructors/{instructorId}/courses
Content-Type: application/json

{
  "title": "Spring Boot from Scratch",
  "description": "Learn Spring Boot with hexagonal architecture",
  "price": 49.99,
  "category": "PROGRAMMING",
  "difficulty": "INTERMEDIATE",
  "durationHours": 20
}
```
New courses are created with status `DRAFT`. Publish them via `PATCH /api/v1/courses/{id}` with `"status": "PUBLISHED"`.

### Enroll Student
```http
POST /api/v1/students/{studentId}/enrollments/{courseId}
```
No request body required. Enrollment starts with `status: ACTIVE` and `progressPercentage: 0`.

### Update Enrollment Progress
```http
PATCH /api/v1/students/{studentId}/enrollments/{courseId}/progress
Content-Type: application/json

{
  "progressPercentage": 100
}
```
When progress reaches 100, the enrollment status automatically transitions to `COMPLETED` and a domain event is published.

### Submit a Review
```http
POST /api/v1/students/{studentId}/enrollments/{courseId}/review
Content-Type: application/json

{
  "rating": 5,
  "comment": "Excellent course, very well structured!"
}
```
Business rules: the student must be enrolled in the course and can only leave one review per course.

### Partial Update (PATCH)
Only send the fields you want to change. At least one field is required — an empty body returns `409 Conflict`.
```http
PATCH /api/v1/courses/{id}
Content-Type: application/json

{
  "status": "PUBLISHED",
  "price": 39.99
}
```

## Features

### Caching (Caffeine)
Course list queries are cached with a 10-minute TTL (`maximumSize=500`). The cache is evicted automatically on any create, update, or delete operation.

### JPA Auditing
All entities automatically track `createdAt` and `updatedAt` timestamps via a shared `Auditable` base class and `@EnableJpaAuditing`.

### Domain Events (async)
- `StudentEnrolledEvent` — published after a student successfully enrolls
- `EnrollmentCompletedEvent` — published when progress reaches 100%
- Listeners run in a separate thread (`@Async`) so the main transaction is not blocked

### Scheduled Tasks
`EnrollmentStatsScheduler` fires daily at midnight (`@Scheduled(cron = "0 0 0 * * *")`).

### Monitoring
Spring Boot Actuator exposes `health`, `info`, and `metrics` at `/actuator/*`.

## Error Responses

The API uses [RFC 7807 Problem Details](https://datatracker.ietf.org/doc/html/rfc7807) for all error responses, enabled via `spring.mvc.problem-details.enabled: true`.

| HTTP Status | When |
|---|---|
| `400 Bad Request` | Invalid request format or constraint violation |
| `404 Not Found` | Resource does not exist |
| `409 Conflict` | Business rule violation (duplicate email, already enrolled, already reviewed, empty update body) |
| `422 Unprocessable Entity` | Bean Validation failure (with field-level error map) |

## Testing

The project has three layers of tests:

### Unit Tests (no infrastructure required)
Mock all outbound ports with Mockito. Cover happy path and error scenarios.
```
student/application/usecase/CreateStudentServiceTest
student/application/usecase/GetStudentServiceTest
enrollment/application/usecase/EnrollStudentServiceTest
```

### Controller Tests (`@WebMvcTest`)
Test the HTTP layer in isolation: serialization, status codes, validation.
```
student/infrastructure/in/web/StudentControllerTest
instructor/infrastructure/in/web/InstructorControllerTest
enrollment/infrastructure/in/web/EnrollmentControllerTest
```

### Integration Tests (requires Docker)
Spin up a real MySQL container via Testcontainers. Test the full persistence stack end-to-end.
All integration tests extend `AbstractPersistenceTest` which configures the container via `@DynamicPropertySource`.
```
student/infrastructure/out/persistence/StudentPersistenceAdapterTest
enrollment/infrastructure/out/persistence/EnrollmentPersistenceAdapterTest
instructor/infrastructure/out/persistence/InstructorPersistenceAdapterTest
course/infrastructure/out/persistence/CoursePersistenceAdapterTest
profile/infrastructure/out/persistence/ProfilePersistenceAdapterTest
```
